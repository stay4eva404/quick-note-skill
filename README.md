## 🤔 为什么需要它

QuickNote 是一个轻量的速记工具：当你发现需要后续处理的事项，但不想立刻切换上下文时，一句话告诉 AI，它就会按规范格式记录下来，并支持状态流转与检索——让你集中查阅、按节奏处理。

工作中经常有这样的瞬间：

| 场景 | 内心独白 |
|------|---------|
| 写代码时发现一个潜在 bug | _"现在正改别的东西，不想切换上下文"_ |
| 产品讨论时冒出一个想法 | _"想记下来，但不打算立刻展开"_ |
| 调研时遇到一个待确认的问题 | _"留到后面再查，先继续"_ |
| 生活里突然想起一件待办 | _"先记下，别打断手头工作"_ |

- 当场处理 → **打断工作流**
- 啥都不做 → **容易忘**

QuickNote 就是给这类「先记下来回头再看」的场景设计的。

## 🚀 快速开始

### 安装

QuickNote 是一个**通用 Agent Skill**，遵循 [Anthropic Agent Skills](https://github.com/anthropics/agent-skills) 规范，可在任何支持 SKILL.md 加载的 AI 编程助手中使用。

#### 安装路径

放入对应平台的全局 skills 目录即可：

| 平台 | 路径 |
|------|------|
| Claude Code / Codex | `~/.claude/skills/` 或 `~/.codex/skills/` |
| Trae / Cursor / OpenCode | `~/.trae/skills/`、`~/.cursor/skills/`、`~/.opencode/skills/` |
| 其他 Agent | 一般为 `~/.<agent>/skills/` |

---

把 SKILL.md 放到对应平台的全局或项目级 skills 目录即可：

```bash
# 1. 克隆仓库
git clone https://github.com/stay4eva/quick-note-skill.git

# 2. 全局安装（以 Claude Code 为例，其他平台替换路径前缀即可）
mkdir -p ~/.claude/skills/quicknote
cp quick-note-skill/SKILL.md ~/.claude/skills/quicknote/SKILL.md

# 3. 或仅当前项目使用
mkdir -p ./.claude/skills/quicknote
cp quick-note-skill/SKILL.md ./.claude/skills/quicknote/SKILL.md
```

安装完成后重启 Agent 会话，skill 自动加载。

> 💡 **通用性说明**：本 skill 仅依赖 SKILL.md 文件本身，不包含任何平台特定 API、MCP 工具或脚本，因此可在任何支持 SKILL 规范的 agent 上运行。不同平台的触发词识别能力可能略有差异，但核心功能（创建/更新/检索/编辑/删除）均可用。

## ✨ 特性

- 🌐 **通用场景** — 不限代码，bug / 想法 / 调研 / 生活 / 灵感皆可记录
- 🔄 **五态状态机** — `待查阅 → 已查阅待解决 → 进行中 → 已解决 / 已忽略`
- 🎯 **五类操作** — 创建 · 更新状态 · 编辑正文 · 检索 · 删除
- 🔨 **批量更新** — 按条件筛选后批量流转（需用户确认）
- 🔍 **模糊匹配** — 无需记文件名，按标题 / 关键词自动定位
- ✏️ **正文可编辑** — 用户明确指令下可改任意字段，AI 不主动改
- 📜 **状态历史** — 每次流转记入 `history`，追溯完整生命周期
- 👤 **创建者溯源** — 记录 `author`（用户）与 `created_by`（AI）
- ⚡ **轻量确认** — 操作完简短反馈即返回，不展开、不追问

## 🔄 状态流转

```
                   ┌──────────────┐
                   │   待查阅     │  ← 新建默认
                   └──────┬───────┘
                          ↓
                   ┌──────────────┐
                   │ 已查阅待解决  │
                   └──────┬───────┘
                          ↓
                   ┌──────────────┐
                   │   进行中     │
                   └──────┬───────┘
                          ↓
              ┌───────────┴───────────┐
              ↓                       ↓
       ┌──────────────┐        ┌──────────────┐
       │   已解决     │        │   已忽略     │
       └──────────────┘        └──────────────┘
```

| 规则 | 说明 |
|------|------|
| ⏩ 允许跳跃 | 任何状态可直接跳到 `已解决` 或 `已忽略` |
| ◀️ 允许回退 | `进行中` 可退回 `已查阅待解决` |
| 🔁 允许重新打开 | `已解决` / `已忽略` 可恢复到任意状态 |
| 🗣️ 口语映射 | "搞定"→`已解决`、"先放着"→`已忽略`、"在做"→`进行中` |

## 💾 存储约定

| 项目类型 | 默认目录 | 说明 |
|:--------:|:--------:|------|
| 任意项目 | `.quicknote/` | 🌟 默认推荐，dot 前缀隐藏不污染视觉 |
| 已有 `docs/quicknote/` | `docs/quicknote/` | ↩️ 向后兼容，自动沿用 |

**文件名规则**

```
格式：YYYY-MM-DD-HHmm_简短标题.md
示例：2026-07-02-1430_登录超时未处理.md
```

- 特殊字符自动清洗：`/ \ : * ? " < > |` 与空格 → 下划线 `_`
- 长度超过 40 字符自动截断
- 同分钟冲突自动追加序号：`_2.md`、`_3.md`

## 📄 文件结构

```markdown
---
type: <bug|优化|技术债|想法|待办|待调研|其他>
status: <待查阅|已查阅待解决|进行中|已解决|已忽略>
priority: <high|medium|low>
scene: <代码|产品|想法|调研|生活|其他>
author: <创建者用户名>
created_by: <agent名称 / 模型名称>
created: <YYYY-MM-DD HH:mm>
updated: <YYYY-MM-DD HH:mm>
tags: []
history: []
---

# <标题>

## 发现背景
<上下文>

## 问题描述
<详细描述>

## 相关文件
- `<文件路径>`           ← 非代码场景可删除本节

## 初步想法
<简要的根因猜想、排查方向、可能的修复思路>

## 备注
<额外说明>
```

### 字段说明

| 字段 | 取值规则 |
|------|---------|
| `author` | 自动取 `git config user.name` → 系统用户名 → `unknown` |
| `created_by` | AI 自行填入，格式 `<agent名称> / <模型名称>`（如 `Trae Agent / GLM-5.2`） |
| `history` | 每次状态更新追加 `{ time, from, to }`，保留完整流转轨迹 |
| `tags` | 默认空数组，不自动提取，由用户明确给出才填入 |
| `created`/`updated` | 用户本地时区，格式 `YYYY-MM-DD HH:mm`，无法确定时用 `Asia/Shanghai` |

## 🎮 使用示例

### 1️⃣ 创建

```
👤 用户：这里登录超时的错误没处理，先记一下，回头再看

🤖 AI  ：已记录到 .quicknote/2026-07-02-1430_登录超时未处理.md 📝
         （状态：待查阅）
```

### 2️⃣ 更新状态

```
👤 用户：登录超时那个我开始处理了

🤖 AI  ：已更新 2026-07-02-1430_登录超时未处理.md
         待查阅 → 进行中 ✅
```

### 3️⃣ 编辑正文

```
👤 用户：把登录超时那个的描述改一下，加上"在弱网环境下复现"

🤖 AI  ：已修改 2026-07-02-1430_登录超时未处理.md：更新问题描述 ✏️
```

> 📌 **编辑原则**：仅在用户明确指令下修改，AI 不主动建议或自行扩写。
> 模糊指令（如"完善一下描述"）会追问一次具体内容，不自行发挥。

### 4️⃣ 模糊匹配多候选

```
👤 用户：把登录那个标记为已解决

🤖 AI  ：找到多个匹配，请选择：
         - [进行中] 2026-07-02-1430_登录超时未处理.md — 登录超时未处理
         - [待查阅] 2026-07-02-1500_登录页样式错位.md  — 登录页样式错位

👤 用户：第一个

🤖 AI  ：已更新 2026-07-02-1430_登录超时未处理.md
         进行中 → 已解决 ✅
```

### 5️⃣ 检索

```
👤 用户：列出未解决的 quicknote

🤖 AI  ：
         - [进行中] 2026-07-02-1430_登录超时未处理.md — 登录超时未处理（medium）
         - [待查阅] 2026-07-02-1505_调研竞品订阅模型.md — 调研竞品订阅模型（low）

         共 2 条（待查阅 1 / 进行中 1）
```

> 📌 **默认只显示未解决**（`待查阅` + `已查阅待解决` + `进行中`）<br>
> 已解决/已忽略需明示「包括已解决」或「看全部」才显示。

### 6️⃣ 批量更新

```
👤 用户：把所有 bug 标记为已忽略

🤖 AI  ：将批量更新为「已忽略」，共 3 条：
         - 2026-07-02-1430_登录超时未处理.md
         - 2026-07-02-1500_登录页样式错位.md
         - 2026-07-02-1510_表单校验异常.md
         确认执行？

👤 用户：确认

🤖 AI  ：已更新 3 条 → 已忽略 ✅
```

### 7️⃣ 删除

```
👤 用户：把表单校验那条删了

🤖 AI  ：确认删除 2026-07-02-1510_表单校验异常.md（表单校验异常）？
         此操作不可撤销。

👤 用户：确认

🤖 AI  ：已删除 2026-07-02-1510_表单校验异常.md 🗑️
```

## 🧱 设计原则

<table>
<tr>
<td width="50%" valign="top">

**⚡ 不打断工作流**

记录与流转都极简，操作完立即返回当前任务。

</td>
<td width="50%" valign="top">

**🚧 AI 不越界**

只负责记录/更新/检索/简短确认，但"初步想法"节应主动给出有价值的根因猜想与分析——这是 quicknote 的核心价值。不展开成完整方案讨论、不追问用户。

</td>
</tr>
<tr>
<td width="50%" valign="top">

**🔒 修改有界限**

所有字段（正文+元数据）均可在用户**明确指令**下修改，AI 不主动建议或自行扩写。`status`/`history` 走状态更新操作。

</td>
<td width="50%" valign="top">

**🗑️ 删除需确认**

支持删除 quicknote 文件，但必须用户明确确认才执行。归档用 `已忽略`，彻底不要才用删除。

</td>
</tr>
<tr>
<td colspan="2" align="center">

**🗂️ 每个项目独立**

不支持跨项目检索，每个项目的 quicknote 存在该项目下，互不干扰。

</td>
</tr>
</table>
